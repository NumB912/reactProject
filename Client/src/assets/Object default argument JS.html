Trong JavaScript, bạn có thể truyền giá trị mặc định cho đối số là object trong hàm như sau:

Ví dụ cơ bản:

js
function sayHello({ name = "Khách", age = 18 } = {}) {
  console.log(`Xin chào ${name}, bạn ${age} tuổi.`);
}

sayHello(); // Xin chào Khách, bạn 18 tuổi.
sayHello({ name: "An" }); // Xin chào An, bạn 18 tuổi.
sayHello({ name: "Bình", age: 25 }); // Xin chào Bình, bạn 25 tuổi.

Giải thích:

  • { name = "Khách", age = 18 } = {}

    • Phần = {} đảm bảo nếu người dùng không truyền tham số vào thì vẫn có một object để destructure.

    • name = "Khách" và age = 18 là giá trị mặc định nếu key đó không được truyền vào.

Một ví dụ khác với object mặc định phức tạp hơn:

js
function configApp(options = { debug: false, theme: "light" }) {
  console.log("Debug mode:", options.debug);
  console.log("Theme:", options.theme);
}

configApp(); // Debug mode: false, Theme: light
configApp({ debug: true }); // Debug mode: true, Theme: undefined

⚠️ Lưu ý: ví dụ này không destructure nên nếu không truyền đủ key, các key còn lại sẽ undefined. Để khắc phục, nên dùng destructuring như ví dụ đầu tiên.

Nếu bạn đang dùng ES6+ thì nên dùng destructuring có giá trị mặc định để code an toàn và dễ đọc hơn.

À, bạn muốn chỉ một key trong object có giá trị mặc định, còn những key khác thì giữ nguyên nếu được truyền vào đúng không?

👉 Đây là cách làm:

✅ Ví dụ: chỉ đặt mặc định cho age, còn name giữ nguyên nếu được truyền vào

js
function sayHello({ name, age = 18 } = {}) {
  console.log(`Xin chào ${name || "Khách"}, bạn ${age} tuổi.`);
}

sayHello(); // Xin chào Khách, bạn 18 tuổi.
sayHello({ name: "An" }); // Xin chào An, bạn 18 tuổi.
sayHello({ age: 25 }); // Xin chào Khách, bạn 25 tuổi.
sayHello({ name: "Bình", age: 30 }); // Xin chào Bình, bạn 30 tuổi.

🧠 Giải thích:

  • ({ name, age = 18 } = {}): destructuring có mặc định cho age, và object đầu vào cũng có mặc định là {} để tránh lỗi khi không truyền gì.

  • name || "Khách" là cách đơn giản để cho name mặc định nếu undefined.

Nếu bạn muốn giữ nguyên object mà chỉ gán mặc định cho 1 key mà không dùng destructuring, có thể làm thế này:

✅ Cách dùng object spread:

js
function sayHello(options = {}) {
  const defaultOptions = { age: 18 };
  const finalOptions = { ...defaultOptions, ...options };

  console.log(`Xin chào ${finalOptions.name || "Khách"}, bạn ${finalOptions.age} tuổi.`);
}

sayHello(); // Xin chào Khách, bạn 18 tuổi.
sayHello({ name: "An" }); // Xin chào An, bạn 18 tuổi.

Nếu bạn chỉ muốn một prop (thuộc tính) trong object đầu vào có giá trị mặc định, thì vẫn dùng được destructuring như sau:

✅ Ví dụ: Chỉ đặt mặc định cho một prop — ví dụ isAdmin

js
function login({ username, isAdmin = false } = {}) {
  console.log(`Username: ${username || "guest"}`);
  console.log(`Admin: ${isAdmin}`);
}

login(); // Username: guest, Admin: false
login({ username: "an" }); // Username: an, Admin: false
login({ username: "hoa", isAdmin: true }); // Username: hoa, Admin: true

✅ Nếu chỉ truyền đúng 1 prop:

Bạn có thể truyền từng prop như thế này:

js
login({ username: "an" }); // isAdmin sẽ dùng mặc định = false
login({ isAdmin: true }); // username không có nên sẽ là "guest"

✅ Nếu chỉ muốn mặc định cho một prop khi dùng function nhận object:

Cách đơn giản và an toàn nhất vẫn là destructuring:

js
function myFunc({ oneProp = "mặc định" } = {}) {
  console.log(oneProp);
}

myFunc(); // "mặc định"
myFunc({ oneProp: "giá trị truyền vào" }); // "giá trị truyền vào"

Bạn đang dùng hàm kiểu nào? Mình có thể viết theo đúng context bạn đang làm.

js
function sayHi({ name = "Khách" } = {}) {

sayHi(); // Xin chào, Khách!
